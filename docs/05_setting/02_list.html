列表设置

列表设置用于配置主窗口内容区域的显示样式，提供灵活的视图选项以满足不同翻译工作流程的需求。合理配置列表显示可以显著提高翻译效率和阅读体验。

索引显示

显示索引列

控制是否在列表中显示索引列。索引列显示每个翻译条目的序号，帮助您：

  • 快速了解当前项目的翻译条目总数
  • 在团队协作时准确定位和引用特定条目
  • 追踪翻译进度和完成度

默认值: 启用
建议: 对于大型项目，建议启用索引列以便更好地管理和定位条目。

索引从 0 开始

设置索引列的起始编号方式：

  • 从 0 开始: 符合程序员习惯，与数组索引保持一致，适合技术文档翻译
  • 从 1 开始: 符合一般用户的计数习惯，更直观自然

默认值: 从 1 开始
适用场景:

  • 技术团队或开发者：推荐从 0 开始
  • 非技术翻译团队：推荐从 1 开始

列显示配置

显示常量列

对于支持常量的strings文件，控制是否显示常量列。

常量列显示每个翻译条目的唯一标识符，便于：

  • 理解代码中的引用关系
  • 进行精准的上下文翻译
  • 维护翻译文件的技术一致性

建议: 对于 iOS/macOS 开发项目，建议启用此选项。

基础语言在第一列

将基础语言（Base Language）固定在列表的最左侧第一列位置。

优势:

  • 翻译时始终能看到原文参考
  • 便于进行逐条对照翻译
  • 减少水平滚动，提高工作效率

默认值: 启用
建议: 强烈推荐启用，特别是在多语言并行翻译场景下。

内容显示

多行显示

控制列表单元格的文本显示方式：

  • 单行模式: 文本超出列宽时显示省略号（...），保持列表紧凑
  • 多行模式: 自动换行显示完整内容，无需悬停或展开

权衡考虑:

  • 单行模式：列表视图紧凑，可同时查看更多条目，但需要悬停查看完整内容
  • 多行模式：可直接查看完整文本，但会增加列表高度，减少可见条目数

默认值: 单行模式
适用场景:

  • 短文本翻译（按钮、标签等）：推荐单行模式
  • 长文本翻译（说明、提示等）：推荐多行模式

最大显示行数

在多行显示模式下，限制单个单元格的最大显示行数。

默认值: 3 行

说明: 此设置可防止单个超长文本（如包含数百或数千字符的法律条款、说明文档）占据过多垂直空间，导致其他条目无法查看。超出最大行数的内容将被截断并显示省略号。

推荐配置:

  • 简短UI文本项目：2-3 行
  • 一般文档项目：3-5 行
  • 长文档项目：5-8 行

搜索高亮

高亮匹配

在执行搜索操作时，是否高亮显示匹配到的关键词部分。

功能优势:

  • 快速识别搜索结果的匹配位置
  • 理解搜索算法的匹配逻辑
  • 提高搜索结果的可读性和精准度

默认值: 启用
建议: 始终启用，特别是在使用模糊搜索或正则表达式搜索时。

高亮颜色

自定义搜索结果中匹配关键词的高亮颜色。

默认值: 系统强调色（通常为蓝色或系统主题色）
建议: 选择与背景对比度高的颜色，确保在不同主题（浅色/深色模式）下都清晰可见。

高亮位置

设置高亮颜色应用的位置：

  • 前景色（Foreground）: 改变文字颜色
  • 背景色（Background）: 改变文字背景色

默认值: 背景色
推荐: 使用背景色高亮通常效果更好，因为可以保持文本颜色的一致性。

显示模式

列表模式

TransX 支持两种列表显示模式，以适应不同的翻译工作流程：

多语言模式（Multi-Language Mode）

在列表中同时显示所有语言列，实现多语言并行对比。

适用场景:

  • 需要同时查看和对比多个语言版本
  • 进行多语言一致性检查
  • 小屏幕设备或语言数量较少的项目（2-4种语言）

优势: 全局视图，方便跨语言对比
劣势: 语言较多时需要频繁水平滚动

单语言模式（Single Language Mode）

只显示选定的单个目标语言列，提供更聚焦的翻译视图。

适用场景:

  • 专注翻译单一目标语言
  • 项目包含大量语言（5种以上）
  • 需要更大的编辑空间

优势: 界面简洁，无需水平滚动，更大的编辑区域
配合: 可结合「显示基础语言列」选项保留原文参考

显示基础语言列

在单语言模式下，是否同时显示基础语言列作为翻译参考。

功能说明: 启用此选项后，界面将显示两列：基础语言 + 当前选择的目标语言。

默认值: 启用
建议: 强烈建议启用，确保翻译时始终有原文参考。

显示语言面板

在单语言模式下，是否显示语言切换面板，用于快速切换查看的目标语言。

面板功能:

  • 快速切换不同目标语言
  • 显示各语言的翻译完成状态
  • 支持语言过滤和搜索（多语言项目）

默认值: 启用
建议: 如果需要频繁切换语言，建议启用；如果只翻译固定语言，可关闭以节省屏幕空间。

列宽设置

列宽

精确控制列表中各列的宽度范围，以优化屏幕空间利用。

配置选项:

  • 自动宽度: 根据内容自动调整列宽
  • 固定宽度: 为每列设置固定像素宽度
  • 最小/最大宽度: 限制列宽的范围

调整方式:

  1. 拖动列标题之间的分隔线直接调整
  2. 在设置中输入精确数值
  3. 双击分隔线自动适应内容宽度

最佳实践:

  • 索引列: 40-60px（根据条目数量）
  • 常量列: 150-250px（根据常量命名长度）
  • 语言列: 200-400px（根据文本平均长度和字符集）
  • 为常用语言分配更多宽度
  • 预留一定空白空间，避免文本过于拥挤

提示: 列宽设置会自动保存，下次打开项目时恢复。
