项目配置

项目配置是 TransX 中针对单个项目的个性化设置中心。与全局偏好设置不同，这里的配置仅影响当前打开的项目，允许您为每个项目定制专属的工作环境和管理策略。

当您管理多个具有不同特点的项目时（例如：不同的编程语言、不同的本地化策略、不同的团队规范），项目配置功能确保每个项目都能使用最适合自己的设置，而不会相互干扰。

配置模块说明

通用

项目的基础设置和行为配置。包括项目元信息、默认行为、自动化选项等。通用设置定义了项目的基本工作模式，影响整体的交互体验和自动化流程。这里的配置涵盖了从文件处理策略到编辑行为的多个方面，是项目配置的核心部分。

高亮显示

视觉辅助和内容标识配置。通过颜色编码和视觉提示，高亮显示特定状态或类型的内容条目。例如：

  • 未翻译的条目
  • 最近修改的内容
  • 包含特殊字符的字符串
  • 超长文本警告
  • 空值或异常数据

合理配置高亮规则可以显著提升内容审查效率，让问题条目一目了然，减少遗漏和错误。

常量

代码常量生成的详细配置。如果项目启用了常量模式，这个模块允许您精细控制常量的生成规则：

  • 常量命名规范和前缀
  • 代码风格和格式化选项
  • 常量文件的组织结构
  • 自动生成的触发时机
  • Objective-C 和 Swift 的适配选项

这些设置直接影响生成的代码质量和团队代码规范的一致性，对于维护代码库的整洁性至关重要。

本地化

语言和本地化策略的专项配置。管理项目支持的语言列表、语言间的关系、本地化工作流程等：

  • 支持的语言列表和优先级
  • 基础语言（参考语言）设置
  • 语言回退（fallback）机制
  • 翻译内容的验证规则
  • 多语言之间的同步策略

本地化配置确保项目的多语言支持符合产品需求和国际化标准，是多语言管理的策略中枢。

文件夹

项目文件结构和路径管理配置。定义 TransX 如何理解和处理项目的目录结构：

  • .strings 文件的搜索路径
  • 常量文件的输出路径
  • 忽略目录和文件的规则
  • 导出内容的默认保存位置
  • 备份文件的存储策略

文件夹配置直接影响 TransX 的文件扫描和输出行为，合理设置可以让工具更好地适应项目的实际结构。

配置的持久化

项目配置保存在项目根目录的 TransX 配置文件中（通常是 .localiztion），并随项目一起管理。这意味着：

  1. 团队共享：配置文件可以提交到版本控制系统，团队成员打开项目时自动应用相同的配置
  2. 环境一致：确保所有开发者使用统一的本地化管理规则和代码生成标准
  3. 项目独立：不同项目的配置互不影响，可以采用完全不同的管理策略

配置的访问方式

  • 菜单栏：操作 → 项目配置
  • 快捷键：根据系统设置的快捷键（command + .）
  • 工具栏：主窗口工具栏的配置图标

最佳实践建议

  1. 首次配置时仔细设置：项目创建后就配置好这些选项，避免后期频繁调整
  2. 遵循团队规范：常量命名、代码风格等应与团队整体规范保持一致
  3. 合理使用高亮：过多的高亮规则会导致视觉干扰，选择最重要的几个即可
  4. 定期审查配置：随着项目发展，适时调整配置以适应新的需求
  5. 文档化特殊配置：如果使用了非标准配置，建议在项目文档中说明原因

通过合理配置这些选项，您可以让 TransX 完美适应项目的特定需求，最大化提升多语言管理的效率和质量。
