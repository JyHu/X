5 分钟学会使用 TransX 管理多语言

TransX 能帮你做什么？

TransX 是专为 Xcode 项目设计的多语言管理工具，让 iOS/macOS 应用的本地化工作变得简单高效。它解决了手动编辑 .strings 文件容易出错、难以维护、缺乏类型安全等问题。

TransX 欢迎界面 - 快速访问最近项目

四步开始使用

1

创建项目

  1. 打开 TransX，点击"新建项目"
  2. 选择你的 Xcode 项目文件夹
  3. （可选）设置搜索目录和忽略规则，排除不需要管理的文件夹（如 Pods）
  4. 点击"检索"，让 TransX 自动扫描所有 .strings 文件

项目选择 - 设置搜索和忽略规则

💡 提示：如果不确定如何设置，保持默认即可，TransX 会扫描整个项目。
2

配置参数

设置两个关键参数：

  • 基础语言：选择你的主要开发语言（如"中文"或"英文"）
    这个语言会作为分组依据和翻译参考
  • 常量模式（推荐开启）：为每个本地化字符串生成类型安全的常量
    避免手写字符串 key 时的拼写错误

参数配置 - 设置基础语言和常量模式

💡 提示：首次使用建议开启"去除空白字符"和"常量模式"，这能避免很多常见问题。
3

编辑内容

进入主工作区，界面分为多个区域：

主工作区 - 左侧导航、中间列表、右侧编辑

左侧导航栏

显示所有 .strings 文件，点击切换

中间列表区

显示所有条目，支持搜索和筛选

右侧编辑器

编辑各语言的内容

常用操作：

  • 新建条目：按 ⌘N 或点击工具栏的"新建"按钮
  • 编辑内容：选中条目，在右侧编辑器修改
  • 查找内容：按 ⌘F 打开搜索框
  • 筛选状态：使用工具栏的筛选按钮（缺失翻译、新建、修改等）
💡 提示：在基础语言文件中使用 // MARK: - 分组名 可以组织条目，让内容更清晰。
4

导出保存

编辑完成后必须导出才能生效：

  1. 点击工具栏的"导出到项目"按钮
  2. TransX 会将所有修改写回 .strings 文件
  3. 如果启用了常量模式，还会生成常量代码文件
  4. 回到 Xcode，刷新项目即可看到更新
⚠️ 重要：如果不导出，所有修改只保存在 TransX 中，不会写入项目文件！

实用技巧

⚡

提高效率

  • 使用筛选器快速找到需要处理的条目
  • 选中多个条目后可以批量操作
  • 记住常用快捷键：⌘N 新建、⌘S 保存、⌘F 搜索
🛡️

避免问题

  • 定期导出，不要积累太多修改
  • 注意 Git 分支切换提醒
  • 删除的条目会进入回收站，可恢复
👥

团队协作

  • 项目配置可提交 Git，自动共享设置
  • 使用导出功能（CSV/JSON）协作
  • 利用常量模式，代码更易 review
🎨

个性化定制

  • 偏好设置：调整界面主题、字体大小和列显示
  • 项目配置：为不同项目设置专属的工作环境
  • 配置高亮规则让问题条目一目了然
