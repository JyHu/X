# 显示选项

![](./imgs/img22.png)

## 功能概述

显示选项是 TransX 中用于精细化控制内容列表视觉呈现的功能模块，提供包括索引列、常量列、语言列顺序和文本换行方式在内的多维度配置能力。通过合理组合这些选项，用户可以根据当前工作任务的特点（翻译、开发、审核等）定制最优的界面布局，实现：

- **信息密度优化**：在有限的屏幕空间内展示最有价值的内容
- **工作流程适配**：为不同角色和工作阶段提供个性化视图
- **可读性提升**：通过合理的列排序和文本展示方式改善内容理解效率

每个显示选项都经过精心设计，既可独立使用，也可组合配置，系统会自动保存用户偏好，在下次打开项目时恢复上次的配置状态。

## 可配置选项

### 1. 索引列

**功能说明**：  
在列表最左侧显示一个自动编号的索引列，为每个可见条目分配一个从 1 开始的连续序号。序号会根据当前的筛选、搜索和排序结果动态更新。

**使用价值**：

1. **快速定位与引用**
   - 在团队讨论中可以通过编号精确指代某个条目（如"请检查第 15 条的德语翻译"）
   - 避免口头描述 Key 值时的冗长和错误
   - 在会议记录或任务分配中作为简洁的引用标识

2. **数量统计**
   - 直观了解当前显示的条目总数
   - 快速评估工作量（如"还有 50 条待翻译"）
   - 验证筛选或搜索结果的数量

3. **批量操作辅助**
   - 便于记录需要处理的条目位置（如"需要修改 10-15 和 28-32"）
   - 在分配任务时可以按编号范围划分（如"A 负责 1-100，B 负责 101-200"）

4. **跟踪与验证**
   - 对照设计文档或测试用例中的条目编号进行内容核对
   - 在导出报告时提供统一的编号体系

**应用场景**：
- 团队会议中讨论具体的本地化条目
- 记录审核意见和修改清单
- 对照外部文档进行逐条核查
- 评估翻译进度和工作量

**视觉示例**：
```
+-------+---------------------------+-------------+-------------+
| Index | Key                       | zh-Hans     | en          |
+-------+---------------------------+-------------+-------------+
|   1   | com.app.login.title       | 登录        | Login       |
|   2   | com.app.login.button      | 立即登录    | Sign In     |
|   3   | com.app.profile.title     | 个人资料    | Profile     |
+-------+---------------------------+-------------+-------------+
```

### 2. 常量列

**功能说明**：  
显示根据偏好设置中的命名规则自动生成的常量名称列。常量是代码中引用本地化字符串的标识符，通常由 Key 值转换而来。

**生成规则**：
- 常量格式由"偏好设置 > 常量配置"中定义的规则决定
- 支持多种命名风格：驼峰命名、帕斯卡命名、下划线命名等
- 可以配置前缀、后缀和特殊字符处理规则

**使用价值**：

1. **代码集成便利**
   - 开发人员可以直接复制常量名称用于代码中
   - 避免手工输入长字符串 Key 造成的错误
   - 提供编译期类型检查，提升代码安全性

2. **命名规范管理**
   - 统一查看和管理常量的命名规则
   - 确保项目中所有常量遵循一致的命名风格
   - 便于代码审查和规范检查

3. **快速查找和定位**
   - 开发人员可以通过常量名称快速定位对应的本地化内容
   - 支持在常量列中搜索，建立代码与翻译的双向索引
   - 在调试时快速确认常量对应的实际文本

4. **重构和维护支持**
   - 在代码重构时，可以批量查看和调整常量名称
   - 检查常量命名是否会产生冲突
   - 生成常量定义文件的参考数据

**应用场景**：

- **开发阶段**：开发人员需要在代码中引用本地化字符串
- **代码审查**：检查常量使用是否规范和一致
- **文档生成**：导出常量列表用于技术文档
- **自动化工具**：为脚本提供常量与 Key 的映射关系

**显示示例**：

```
+---------------------------+----------------------+-------------+
| Key                       | 常量                  | zh-Hans     |
+---------------------------+----------------------+-------------+
| com.app.login.title       | kLoginTitle          | 登录        |
| com.app.login.button      | kLoginButton         | 立即登录    |
| com.app.profile.settings  | kProfileSettings     | 设置        |
+---------------------------+----------------------+-------------+
```

**注意事项**：
- 常量列的内容完全基于配置的生成规则
- 如果修改了生成规则，需要重新生成常量才会更新显示
- 某些特殊 Key 可能无法生成合法的常量名，会显示警告或占位符

### 3. 基础语言在第一列

**功能说明**：  
将项目设定的基础语言列固定显示在所有语言列的最左侧（Key 列之后），无论项目中配置了多少种语言，基础语言始终保持在第一列的位置。

**基础语言的定义**：
- 基础语言是项目中被设定为"源语言"或"参考语言"的语言
- 通常是开发团队最熟悉的语言（如中文或英文）
- 在"偏好设置 > 基础语言"中配置

**使用价值**：

1. **翻译对照便利**
   - 翻译其他语言时，可以更方便地对照基础语言的原文
   - 基础语言始终可见，无需横向滚动查找
   - 符合"左侧原文、右侧译文"的传统翻译习惯

2. **视觉固定性**
   - 在多语言横向滚动时，基础语言保持在固定位置
   - 避免在浏览其他语言时失去原文参考
   - 提供稳定的视觉锚点

3. **工作效率提升**
   - 减少视线在屏幕上的移动距离
   - 降低翻译时的认知负担
   - 特别适合需要频繁对照原文的翻译场景

4. **质量保证**
   - 审核人员可以始终对照基础语言检查其他语言的翻译质量
   - 确保翻译不偏离原文的含义和语气
   - 及时发现翻译遗漏或错误

**列顺序对比**：

**未开启时的列顺序**（按语言代码字母顺序排列）：
```
+---------------------------+------+------+------+------+---------+---------+
| Key                       | de   | en   | fr   | ja   | zh-Hans | zh-Hant |
+---------------------------+------+------+------+------+---------+---------+
```

**开启后的列顺序**（假设 zh-Hans 为基础语言）：
```
+---------------------------+---------+------+------+------+------+---------+
| Key                       | zh-Hans | de   | en   | fr   | ja   | zh-Hant |
+---------------------------+---------+------+------+------+------+---------+
```

**应用场景**：

- **基础语言为中文的项目**：中文翻译人员将中文翻译为英语、日语等其他语言
- **基础语言为英文的项目**：国际化团队以英文为源，翻译为各地区语言
- **多语言审核**：审核人员需要对照基础语言检查所有语言的翻译质量
- **翻译记忆构建**：翻译工具需要识别源语言和目标语言的配对

**最佳实践**：
- 在"偏好设置"中正确配置基础语言
- 如果团队有明确的翻译参考语言，建议开启此选项
- 对于没有明确基础语言的项目，可以关闭此选项保持默认顺序

### 4. 多行显示

**功能说明**：  
控制列表中每个单元格内容的换行显示方式，决定文本超出列宽时的呈现形式。

**两种显示模式**：

1. **单行显示模式（默认）**
   - 内容超出列宽时会被截断
   - 显示省略号（...）表示内容未完全显示
   - 单元格高度保持固定，列表更紧凑
   - 一屏可以显示更多的条目

2. **多行显示模式**
   - 单元格高度自动扩展以适应完整内容
   - 文本自动换行，显示所有内容
   - 每个单元格根据实际内容调整高度
   - 可以直接查看完整文本，无需额外操作

**使用价值**：

1. **完整内容可见性**
   - 无需调整列宽即可查看完整的翻译文本
   - 特别适合包含段落文字或较长句子的本地化内容
   - 避免来回调整列宽的繁琐操作

2. **长文本优化**
   - 对于包含多句话的提示信息、说明文本或帮助内容
   - 完整显示避免理解断层
   - 便于检查文本的完整性和连贯性

3. **多语言对比效率**
   - 在对比不同语言的长文本时，可以完整显示而不需要逐个点开查看
   - 直观观察不同语言在文本长度上的差异
   - 发现潜在的 UI 布局问题（如某语言翻译过长）

4. **校对和审核效率**
   - 减少查看完整内容所需的额外操作
   - 可以连续浏览，不被截断打断阅读节奏
   - 更容易发现语法错误、标点问题或格式异常

5. **特殊字符检测**
   - 多行显示时更容易发现隐藏的换行符
   - 识别文本中的不可见字符或格式控制符
   - 检查文本是否包含意外的空格或制表符

**视觉效果对比**：

**单行显示模式**：
```
+---------------------------+----------------------+
| Key                       | zh-Hans              |
+---------------------------+----------------------+
| app.welcome.message       | 欢迎使用我们的应...   |
| app.error.network         | 网络连接失败，请...   |
+---------------------------+----------------------+
```

**多行显示模式**：
```
+---------------------------+----------------------------------+
| Key                       | zh-Hans                          |
+---------------------------+----------------------------------+
| app.welcome.message       | 欢迎使用我们的应用！我们致力于   |
|                           | 为您提供最优质的服务体验。       |
+---------------------------+----------------------------------+
| app.error.network         | 网络连接失败，请检查您的网络设   |
|                           | 置后重试。                       |
+---------------------------+----------------------------------+
```

**适用场景**：

**开启多行显示**：
- 项目包含大量长文本（如帮助文档、法律声明、使用说明）
- 需要校对完整句子的语法和标点
- 对比不同语言在长度上的差异以评估 UI 适配性
- 检查文本格式问题或特殊字符

**保持单行显示**：
- 项目主要包含短文本（如按钮标签、菜单项、简短提示）
- 需要在一屏内浏览尽可能多的条目
- 快速扫描和定位特定条目
- 性能优先（大型项目中多行显示可能影响滚动性能）

**性能考虑**：

⚠️ **注意事项**：
- 多行显示会增加列表行高，减少屏幕上同时显示的条目数量
- 对于包含数千条记录且大量长文本的项目，可能会影响滚动性能
- 在低配设备或大型项目中，建议按需启用
- 可以根据当前任务动态切换单行/多行显示模式

**最佳实践**：

1. **按需切换**：根据当前工作任务选择合适的显示模式
2. **分类处理**：对不同类型的内容（短文本 vs 长文本）采用不同策略
3. **性能优先**：在性能受限时优先使用单行显示
4. **审核阶段**：在质量审核阶段开启多行显示以查看完整内容

## 配置方法

### 通过右键菜单配置

1. 在内容列表区域点击右键
2. 找到"显示"子菜单
3. 勾选或取消勾选需要的显示选项
4. 设置立即生效，无需刷新或重启

### 配置持久化

- TransX 会自动保存您的显示偏好设置
- 下次打开项目时会自动应用上次的配置
- 不同项目可以有独立的显示配置

## 推荐配置方案

### 翻译工作配置

**场景**：专注于翻译工作，需要对照原文
**推荐设置**：
- ✅ 索引列：方便标记问题条目
- ❌ 常量列：翻译人员通常不需要关注
- ✅ 基础语言在第一列：便于对照原文
- ✅ 多行显示：查看完整翻译内容

### 开发集成配置

**场景**：开发人员需要查找和复制常量名称
**推荐设置**：
- ❌ 索引列：开发通常通过搜索定位
- ✅ 常量列：需要复制到代码中使用
- ❌ 基础语言在第一列：可能更关注特定语言
- ❌ 多行显示：代码中通常使用简短的文本

### 质量审核配置

**场景**：QA 审核所有语言的翻译质量
**推荐设置**：
- ✅ 索引列：记录问题条目编号
- ❌ 常量列：审核内容而非代码
- ✅ 基础语言在第一列：对照原文检查
- ✅ 多行显示：检查完整内容

### 快速浏览配置

**场景**：快速浏览大量条目，查找特定内容
**推荐设置**：
- ❌ 索引列：节省空间
- ❌ 常量列：节省空间
- ❌ 基础语言在第一列：使用默认排序
- ❌ 多行显示：提高显示密度，一屏看更多内容

## 注意事项

1. **列宽调整**：开启或关闭某些列后，可能需要手动调整列宽以获得最佳显示效果
2. **性能影响**：在包含数千条记录的大型项目中，开启"多行显示"可能会影响滚动性能
3. **打印导出**：显示选项会影响导出内容的格式，导出前请确认当前配置
4. **屏幕分辨率**：在小屏幕设备上，建议隐藏不必要的列以获得更好的显示效果
5. **基础语言**：确保在"偏好设置"中正确配置了基础语言，否则"基础语言在第一列"选项可能无法按预期工作
6. **常量配置**：常量列的内容取决于"偏好设置 > 常量"中的配置规则，如果常量显示不正确，请检查该配置
